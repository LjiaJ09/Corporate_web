# 企业官网静态站点服务器快速部署手册

适用场景：

1. 项目是纯静态网站
2. 服务器是 Ubuntu
3. 使用 Nginx 部署
4. 本地已有完整项目 zip 包

本次项目的关键特点：

1. 这是静态站点，不需要 `npm install`、`npm run build`
2. 必须上传整个项目，不能只传部分文件
3. 项目里有相对路径和中文目录，目录结构不能乱改
4. 如果 zip 解压后多包一层目录，`nginx root` 要指向真正包含 `index.html` 的目录

---

## 1. 连接服务器

```bash
ssh root@你的域名或公网IP
```

作用：连接到服务器。

---

## 2. 安装 Nginx 和 unzip

```bash
apt update
apt install -y nginx unzip
```

作用：更新软件源并安装 Web 服务器和解压工具。

检查 Nginx 是否启动：

```bash
systemctl status nginx --no-pager
```

作用：查看 Nginx 状态，正常应为 `active (running)`。

---

## 3. 本地上传 zip 到服务器

在本地电脑执行，不是在服务器执行：

```powershell
scp "D:\pythonProject-all\Corporate_web.zip" root@你的域名或公网IP:/root/
```

作用：把本地 zip 上传到服务器 `/root/` 目录。

如果你的 zip 路径或文件名不同，替换成自己的实际路径。

上传后，在服务器检查：

```bash
ls /root
```

作用：确认 zip 已经上传成功。

---

## 4. 创建站点目录并解压

```bash
mkdir -p /var/www/corporate_web
unzip /root/Corporate_web.zip -d /var/www/corporate_web
```

作用：

1. 创建网站目录
2. 把 zip 解压到网站目录

查看解压结果：

```bash
ls /var/www/corporate_web
```

作用：确认文件已经解压出来。

---

## 5. 找到真正的站点根目录

```bash
find /var/www/corporate_web -maxdepth 2 -name "index.html"
```

作用：找到真正包含 `index.html` 的目录。

这一步非常关键。

如果结果是：

```text
/var/www/corporate_web/index.html
```

那么 `nginx root` 就写：

```text
/var/www/corporate_web
```

如果结果是：

```text
/var/www/corporate_web/Corporate_web/index.html
```

那么 `nginx root` 就必须写：

```text
/var/www/corporate_web/Corporate_web
```

这次实际部署就是第二种情况。

---

## 6. 创建 Nginx 站点配置

```bash
nano /etc/nginx/sites-available/corporate_web
```

作用：创建并编辑站点配置文件。

把下面内容完整粘贴进去：

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/corporate_web/Corporate_web;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(jpg|jpeg|png|gif|webp|svg|ico|css|js|pdf|docx|xlsx)$ {
        expires 7d;
        add_header Cache-Control "public";
        access_log off;
    }
}
```

作用：

1. 监听 80 端口
2. 把网站根目录指向正确位置
3. 允许访问 HTML、CSS、JS、图片、PDF、文档等静态资源

注意：

如果你下次部署时 `find` 查到的真实目录不同，就把：

```nginx
root /var/www/corporate_web/Corporate_web;
```

替换成实际目录。

保存 `nano` 的操作：

1. `Ctrl + O`
2. 回车
3. `Ctrl + X`

---

## 7. 启用站点并关闭默认站点

```bash
ln -sf /etc/nginx/sites-available/corporate_web /etc/nginx/sites-enabled/corporate_web
rm -f /etc/nginx/sites-enabled/default
```

作用：

1. 启用新站点配置
2. 删除默认站点，避免默认配置干扰

---

## 8. 检查 Nginx 配置并重载

```bash
nginx -t
```

作用：检查配置语法是否正确。

如果结果类似下面这样，说明配置没问题：

```text
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
```

然后执行：

```bash
systemctl reload nginx
systemctl status nginx --no-pager
```

作用：

1. 重载配置
2. 再确认 Nginx 正常运行

---

## 9. 本机验证网站是否已经部署成功

验证首页响应头：

```bash
curl -I http://127.0.0.1/
```

作用：确认首页是否返回 `200 OK`。

验证首页内容：

```bash
curl http://127.0.0.1/ | head
```

作用：查看首页 HTML 开头。

验证 CSS：

```bash
curl -I http://127.0.0.1/styles.css
```

作用：确认样式文件可访问。

验证图片：

```bash
curl -I http://127.0.0.1/assets/optimized/home/logo.png
```

作用：确认静态图片可访问。

验证端口监听：

```bash
ss -lntp | grep :80
```

作用：确认 Nginx 正在监听 80 端口。

---

## 10. 公网访问验证

先确认你访问的是服务器的公网 IP，不是内网 IP。

浏览器访问：

```text
http://你的公网IP/
```

如果域名已经解析到这台服务器，也可以访问：

```text
http://你的域名/
```

建议继续验证：

```text
http://你的公网IP/about.html
http://你的公网IP/product-detail.html?code=01
http://你的公网IP/product-detail.html?code=02
```

作用：确认首页、关于页、详情页都能打开。

---

## 11. 防火墙和安全组检查

检查 Ubuntu 防火墙：

```bash
ufw status
```

作用：确认系统防火墙是否启用。

如果启用了，放行 80 和 443：

```bash
ufw allow 80/tcp
ufw allow 443/tcp
ufw reload
ufw status
```

作用：放行 HTTP 和 HTTPS 端口。

阿里云安全组还要确认有入方向规则：

1. `TCP 22/22`
2. `TCP 80/80`
3. `TCP 443/443`

授权对象通常写：

```text
0.0.0.0/0
```

---

## 12. 如果公网打不开，按这个顺序排查

先看本机是否正常：

```bash
curl -I http://127.0.0.1/
```

如果本机正常但公网不通，优先排查：

1. 是否访问了公网 IP，而不是内网 IP
2. 阿里云安全组是否放行 80
3. `ufw` 是否拦截
4. `nginx root` 是否写错

查看日志：

```bash
tail -n 50 /var/log/nginx/access.log
tail -n 50 /var/log/nginx/error.log
```

作用：

1. 看请求有没有打到服务器
2. 看 Nginx 有没有报错

---

## 13. 本次实际部署的关键结论

1. 这是静态站点，不需要构建命令
2. zip 原样上传是对的
3. 解压后多包了一层目录，所以最终 `root` 不是：

```text
/var/www/corporate_web
```

而是：

```text
/var/www/corporate_web/Corporate_web
```

4. 服务器本机已验证通过时，如果公网还打不开，大概率是安全组或公网 IP 问题，不是项目本身问题

## 14. 一套最短可复用命令顺序

下面这套命令适合快速重做，执行前记得替换 zip 文件名，并根据 `find` 结果修改 `root`：

```bash
apt update
apt install -y nginx unzip
mkdir -p /var/www/corporate_web
unzip /root/Corporate_web.zip -d /var/www/corporate_web
find /var/www/corporate_web -maxdepth 2 -name "index.html"
nano /etc/nginx/sites-available/corporate_web
ln -sf /etc/nginx/sites-available/corporate_web /etc/nginx/sites-enabled/corporate_web
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
systemctl status nginx --no-pager
curl -I http://127.0.0.1/
curl -I http://127.0.0.1/styles.css
curl -I http://127.0.0.1/assets/optimized/home/logo.png
ss -lntp | grep :80
ufw status
```

本地上传 zip 的命令：

```powershell
scp "D:\pythonProject-all\Corporate_web.zip" root@你的域名或公网IP:/root/
```
